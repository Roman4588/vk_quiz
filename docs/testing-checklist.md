# Чек-лист тестирования

## Авторизация

- [ ] Пользователь может зарегистрироваться как организатор.
- [ ] Пользователь может зарегистрироваться как участник.
- [ ] Пользователь может войти по email и паролю.
- [ ] Неверный пароль не пропускает пользователя.
- [ ] После выхода токен удаляется из браузера.

## Организатор

- [ ] Организатор видит список своих квизов.
- [ ] Организатор может создать квиз.
- [ ] Организатор может добавить текстовый вопрос.
- [ ] Организатор может добавить вопрос с изображением.
- [ ] Организатор может задать одиночный выбор.
- [ ] Организатор может задать множественный выбор.
- [ ] Организатор может запустить комнату.
- [ ] Код комнаты отображается крупно и понятно.
- [ ] Организатор может начать квиз.
- [ ] Организатор может закрыть вопрос вручную.
- [ ] Организатор может перейти к следующему вопросу.
- [ ] Организатор может завершить квиз.

## Участник

- [ ] Участник может подключиться по коду комнаты.
- [ ] Участник видит текущий вопрос.
- [ ] Участник может выбрать один ответ в одиночном вопросе.
- [ ] Участник может выбрать несколько ответов в множественном вопросе.
- [ ] Участник не может отправить пустой ответ.
- [ ] Участник не может ответить после закрытия вопроса.
- [ ] Участник видит правильный ответ после закрытия вопроса.

## Realtime

- [ ] Вопрос появляется у всех подключённых участников без перезагрузки.
- [ ] Лидерборд обновляется после закрытия вопроса.
- [ ] После завершения квиза всем показывается итоговый результат.

## История

- [ ] История проведённых квизов появляется у организатора.
- [ ] История участия появляется у участника.
- [ ] Результаты сохраняются после перезапуска страницы.
