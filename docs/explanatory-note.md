# Краткая пояснительная записка

## 1. Цель проекта

Цель проекта — разработать MVP веб-приложения для проведения квизов в реальном времени. Приложение должно позволять организатору создавать квиз, запускать комнату по коду, показывать вопросы участникам синхронно и автоматически считать результаты.

## 2. Выбор средств разработки

Для клиентской части выбран React, потому что он подходит для создания интерактивного интерфейса с большим количеством состояний: авторизация, создание вопросов, подключение к комнате, ответы, лидерборд. Для сборки используется Vite, так как он быстро запускает проект и удобен для MVP.

Для серверной части выбран Node.js с Express. Express отвечает за REST API: регистрацию, авторизацию, создание квизов, запуск комнат и получение истории. Для обмена событиями в реальном времени используется Socket.IO. Этот инструмент подходит для синхронного показа вопросов, отправки ответов и обновления лидерборда без перезагрузки страницы.

Для хранения данных в MVP используется JSON-файл на сервере. Он сохраняет пользователей, токены, квизы и результаты. Такой вариант проще для учебного прототипа. В промышленной версии его можно заменить на PostgreSQL, SQLite или MongoDB без изменения основной логики приложения.

## 3. Основные роли пользователей

В системе есть две роли:

1. Организатор — создаёт квизы, настраивает правила, запускает комнату и управляет вопросами.
2. Участник — подключается к комнате по коду, отвечает на вопросы и видит лидерборд.

Разделение ролей реализовано на сервере: участник не может создавать квизы и управлять комнатой, а организатор не может отправлять ответы как участник.

## 4. Реализованная функциональность

В MVP реализованы следующие функции:

- регистрация и вход пользователей;
- создание квиза с названием, категорией, правилами и временем на ответ;
- добавление текстовых вопросов;
- добавление вопросов с изображением по URL;
- одиночный и множественный выбор ответа;
- запуск комнаты по короткому коду;
- подключение участников к комнате;
- синхронный показ вопроса всем участникам через WebSocket;
- запрет ответа после закрытия вопроса или окончания таймера;
- подсчёт баллов;
- бонус за скорость ответа;
- лидерборд;
- сохранение истории проведённых и пройденных квизов.

## 5. Сценарий работы приложения

1. Пользователь регистрируется или входит в аккаунт.
2. Если пользователь является организатором, он создаёт квиз.
3. Организатор запускает комнату и получает код.
4. Участник входит в аккаунт и вводит код комнаты.
5. Организатор нажимает кнопку запуска квиза.
6. Первый вопрос появляется у всех подключённых участников.
7. Участник выбирает ответ и отправляет его до окончания времени.
8. Организатор закрывает вопрос или вопрос закрывается автоматически по таймеру.
9. Система считает баллы и обновляет лидерборд.
10. После последнего вопроса квиз завершается, а результат сохраняется в истории.

## 6. Подсчёт баллов

За каждый полностью правильный ответ участник получает 100 баллов. Дополнительно начисляется бонус за скорость: чем больше времени осталось до конца вопроса, тем выше бонус. Для вопроса с множественным выбором ответ засчитывается только при точном совпадении выбранных вариантов с набором правильных вариантов.

## 7. Итог

Получен работоспособный MVP, соответствующий требованиям проекта. Приложение показывает ключевую механику квиза в реальном времени: создание, запуск, подключение участников, синхронный показ заданий, сбор ответов, подсчёт баллов и сохранение результата. Прототип можно развивать дальше: добавить загрузку изображений на сервер, полноценную базу данных, деплой, расширенную аналитику и административную панель.
