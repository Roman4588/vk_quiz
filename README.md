# Quiz Realtime MVP

Работоспособный прототип веб-приложения для проведения квизов в реальном времени.

## Что реализовано

- регистрация и авторизация пользователей;
- две роли: `organizer` и `participant`;
- создание квиза организатором;
- вопросы разных типов: текстовый вопрос и вопрос с изображением по URL;
- одиночный и множественный выбор ответа;
- настройка категории, правил и времени на вопрос;
- запуск комнаты по короткому коду;
- подключение участников к активному квизу;
- показ текущего вопроса всем участникам через WebSocket;
- ограничение ответа временем демонстрации вопроса;
- подсчёт баллов с бонусом за скорость;
- лидерборд после вопросов и после завершения квиза;
- личный кабинет с историей проведённых и пройденных квизов;
- сохранение данных в JSON-хранилище на сервере как MVP-замена БД.

## Стек

- Frontend: React + Vite + CSS
- Backend: Node.js + Express
- Realtime: Socket.IO
- Хранение данных MVP: JSON-файл `server/data/db.json`
- Для промышленной версии рекомендуется заменить JSON-хранилище на PostgreSQL или SQLite.

## Как запустить

Требуется Node.js 18+.

```bash
npm install
npm run dev
```

После запуска:

- клиент: http://localhost:5173
- сервер: http://localhost:4000

## Демо-аккаунты

При первом запуске сервер создаёт тестовые аккаунты:

| Роль | Email | Пароль |
|---|---|---|
| Организатор | org@demo.ru | demo1234 |
| Участник | user@demo.ru | demo1234 |

## Быстрый сценарий проверки

1. Откройте приложение в двух вкладках браузера.
2. В первой вкладке войдите как организатор `org@demo.ru / demo1234`.
3. Создайте квиз или используйте демо-квиз.
4. Нажмите «Запустить комнату» и скопируйте код.
5. Во второй вкладке войдите как участник `user@demo.ru / demo1234`.
6. Введите код комнаты.
7. Организатор нажимает «Начать квиз».
8. Участник отвечает на вопросы.
9. Организатор завершает квиз, всем показывается лидерборд.
